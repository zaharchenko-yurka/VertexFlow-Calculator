## Research Task 1: Circle Intersection Algorithm

**Question**: Как найти точку пересечения двух окружностей для определения новой вершины контура, и что делать если окружности не пересекаются?

### Decision

Использовать стандартный аналитический метод нахождения пересечения двух окружностей с fallback-стратегией постепенного увеличения радиусов.

**Алгоритм**:
1. Даны две окружности:
   - Окружность 1: центр `C1 = (x1, y1)`, радиус `r1`
   - Окружность 2: центр `C2 = (x2, y2)`, радиус `r2`

2. Вычислить расстояние между центрами: `d = sqrt((x2-x1)² + (y2-y1)²)`

3. Проверить условия пересечения:
   - Если `d > r1 + r2` — окружности не пересекаются (слишком далеко)
   - Если `d < |r1 - r2|` — одна окружность внутри другой (нет пересечения)
   - Если `d ≈ 0` и `r1 ≈ r2` — окружности совпадают (бесконечно много точек)

4. Если пересечение найдено — вычислить точку `P = (px, py)` через формулы:
   ```
   a = (r1² - r2² + d²) / (2*d)
   h = sqrt(r1² - a²)
   P2 = C1 + a*(C2-C1)/d  // промежуточная точка
   P = P2 ± h*(C2-C1)⊥/d  // две точки пересечения
   ```
   Выбрать точку, сохраняющую ориентацию контура (по часовой стрелке).

5. **Fallback**: Если пересечения нет (`r1² - a² < 0`), постепенно увеличивать оба радиуса на малую величину `ε = 1 мм` до тех пор, пока не появится пересечение. Максимум итераций: 100 (100 мм суммарного увеличения).

### Rationale

- Аналитический метод обеспечивает точность до 1 мм (требование spec)
- Fallback с постепенным увеличением радиусов гарантирует нахождение решения даже для вырожденных случаев
- Итеративный подход прост в реализации и отладке

### Alternatives Considered

| Alternative | Why Rejected |
|-------------|--------------|
| Пересечение прямых (продление сегментов) | Не сохраняет требуемую длину приращения, нарушает геометрию угла |
| Перпендикулярный офсет от биссектрисы | Сложнее в реализации, менее интуитивно для производственников |
| Пропуск угла при отсутствии пересечения | Нарушает требование непрерывности обработки (FR-006.1) |

## Research Task 2: GLC Encoding (windows-1251, CRLF)

**Question**: Как существующий парсер проекта обрабатывает кодировку и окончания строк GLC-файлов?

### Decision

Следовать спецификации из `.docs/GLC_FORMAL_SPECIFICATION/`:
- **Кодировка**: windows-1251 (Cyrillic) без BOM
- **Окончания строк**: CRLF (`\r\n`)
- **Парсинг**: Использовать `TextDecoder('windows-1251')` в браузере
- **Экспорт**: Использовать `TextEncoder('windows-1251')` с явным добавлением `\r\n`

**Подтверждение из GLC_FIELD_MATRIX.md**:
- Все образцы `.glc` используют windows-1251
- Координаты и числовые значения — FLOAT с точкой (не запятой)
- Метрические экспорты (GVals) используют запятые для украинской локализации

### Rationale

- windows-1251 — отраслевой стандарт GLC для 1С-интеграции
- CRLF обеспечивает совместимость с Windows-инструментами
- Браузерные API `TextDecoder`/`TextEncoder` поддерживают windows-1251

### Implementation Notes

```javascript
// Импорт
const decoder = new TextDecoder('windows-1251');
const text = decoder.decode(arrayBuffer);
const lines = text.split(/\r\n/);

// Экспорт
const encoder = new TextEncoder('windows-1251'); // Требуется полифилл
const lines = [...];
const text = lines.join('\r\n');
const arrayBuffer = encoder.encode(text);
```

**Внимание**: `TextEncoder` не поддерживает windows-1251 в стандартной реализации. Требуется библиотека (например, `iconv-lite` или собственный encoder).

### Alternatives Considered

| Alternative | Why Rejected |
|-------------|--------------|
| UTF-8 с BOM | Не совместим с 1С и существующими GLC-файлами |
| UTF-8 без BOM | Не совместим с отраслевым стандартом |
| ISO-8859-5 | Устаревшая кодировка, не используется в 1С |

## Research Task 3: SVG Pan/Zoom Pattern

**Question**: Как реализовать панорамирование и зумирование SVG на vanilla JS для предпросмотра контуров?

### Decision

Использовать SVG-трансформацию через `viewBox` с обработкой событий мыши.

**Алгоритм**:
1. **Зум колесом мыши**:
   - Перехватить `wheel` event на SVG-контейнере
   - Изменить масштаб: `scale *= event.deltaY > 0 ? 0.9 : 1.1`
   - Обновить `viewBox` с центром на позиции курсора

2. **Пан перетаскиванием**:
   - На `mousedown` запомнить начальную позицию
   - На `mousemove` сместить `viewBox.x` и `viewBox.y`
   - На `mouseup` отпустить

3. **Ограничения**:
   - Минимальный зум: 0.1x
   - Максимальный зум: 10x
   - Панорамирование только в пределах SVG

### Rationale

- `viewBox` — нативный SVG-механизм, не требует внешних библиотек
- Обработка wheel/mouse событий на vanilla JS проста и эффективна
- Соответствует требованию FR-003 (пан/зум колесом мыши)

### Alternatives Considered

| Alternative | Why Rejected |
|-------------|--------------|
| CSS transform: scale/translate | Требует пересчёта координат при экспорте, сложнее с viewBox |
| Библиотека (svg-pan-zoom) | Избыточная зависимость для простого функционала |
| Canvas вместо SVG | Теряется векторная природа, сложнее с экспортом в GLC |
