# Quickstart: Спецраскрой внутренних углов (.glc)

**Date**: 2026-03-15 | **Branch**: `003-special-cut-glc-3` | **Spec**: [spec.md](./spec.md)

---

## Overview

Этот документ описывает шаги по запуску и тестированию функции спецраскроя внутренних углов в калькуляторе VertexFlow.

---

## Prerequisites

1. **WordPress 5.0+** установлен локально или на тестовом сервере
2. **Плагин VertexFlow Calculator** активирован
3. **Тестовые GLC-файлы** из `.docs/GLC_FORMAL_SPECIFICATION/`:
   - `033271.glc` (простая комната, без дуг)
   - `033273.glc` (сложная комната, с дугами и вырезом)

---

## Step 1: Установка плагина

```bash
# Скопировать плагин в директорию WordPress
cp -r vertexflow-calculator/ /path/to/wordpress/wp-content/plugins/

# Активировать плагин через WP-CLI или админ-панель
wp plugin activate vertexflow-calculator
```

---

## Step 2: Встраивание калькулятора

Добавить шорткод на страницу WordPress:

```
[vertexflow_calculator]
```

Или через PHP-шаблон:

```php
<?php echo do_shortcode('[vertexflow_calculator]'); ?>
```

---

## Step 3: Загрузка тестового файла

1. Открыть страницу с калькулятором в браузере
2. Нажать кнопку **"Завантажити .glc"** или перетащить файл в drop-zone
3. Выбрать тестовый файл (например, `033273.glc`)

**Ожидаемый результат**:
- Файл загружен
- Контур(ы) отображены в SVG-превью
- Доступно панорамирование (перетаскивание) и зумирование (колесо мыши)

---

## Step 4: Запуск обработки

1. Убедиться, что чекбокс **"Не обробляти колони і простінки"** установлен (по умолчанию)
2. Нажать кнопку **"Обробити"**

**Ожидаемый результат**:
- Найдены внутренние углы (>180°)
- Применены трансформации согласно правилам (FR-005)
- SVG-превью обновлено с новыми координатами вершин
- В консоль выведены логи обработки (FR-011)

---

## Step 5: Проверка логов

Открыть консоль разработчика (F12) и проверить вывод:

```javascript
// Пример лога
{
  timestamp: "2026-03-15T10:30:00.000Z",
  fileName: "033273.glc",
  totalContours: 1,
  totalVertices: 11,
  results: [
    {
      contourId: "room-1",
      wasModified: true,
      internalAnglesFound: 2,
      internalAnglesProcessed: 2,
      transformations: [...]
    }
  ],
  durationMs: 145
}
```

**Опционально**: Экспорт лога в JSON через кнопку **"Експорт логу"**.

---

## Step 6: Экспорт результата

1. Нажать кнопку **"Завантажити"**
2. Сохранить файл `033273_special_cut.glc`

**Проверка**:
- Открыть файл в текстовом редакторе (Notepad++)
- Проверить кодировку: windows-1251
- Проверить окончания строк: CRLF
- Проверить наличие суффикса `_special_cut` в имени файла (внутри файла не меняется)
- Проверить обновлённые координаты в `POINTS` блоке
- Проверить `StretchParamPer: 10`

---

## Step 7: Валидация геометрии

### Ручная проверка

1. Открыть оригинальный и модифицированный GLC в текстовом редакторе
2. Сравнить блоки `POINTS`:
   - Количество вершин должно увеличиться (если были трансформации)
   - Имена вершин упорядочены (A, B, C...)
3. Сравнить блоки `otrlist_`:
   - Сегменты разделены согласно правилам
   - `JValue` обновлён для модифицированных сегментов

### Автоматическая проверка (рекомендуется)

Сравнить метрики в `GVals`:

```
# Оригинал
A 22.57    # площадь
D 19.565   # периметр

# После обработки
A 22.XX    # площадь (может измениться)
D 19.XXX   # периметр (увеличился на приращения)
```

## Troubleshooting

### Ошибка: "INVALID_GLC_STRUCTURE: Missing ROOMBEGIN block"

**Причина**: Файл не является валидным GLC

**Решение**: Проверить источник файла, использовать тестовые файлы из `.docs/GLC_FORMAL_SPECIFICATION/`

---

### Ошибка: "ENCODING_ERROR: Не удалось декодировать файл"

**Причина**: Файл в неправильной кодировке (не windows-1251)

**Решение**: Конвертировать файл в windows-1251 через Notepad++ или `iconv`

---

### Предупреждение: "Контур имеет некорректную геометрию"

**Причина**: Меньше 3 вершин в контуре

**Решение**: Проверить файл, это минимальная валидация (FR-002.1) — обработка продолжится

---

### Логи не выводятся в консоль

**Причина**: Консоль разработчика не открыта

**Решение**: Нажать F12 для открытия DevTools, обновить страницу
