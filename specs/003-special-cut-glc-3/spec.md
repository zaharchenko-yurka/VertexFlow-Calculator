# Спецификация фичи: Спецраскрой внутренних углов (.glc)

## User Scenarios & Testing *(обязательно)*

### User Story 1 - Загрузка и обработка контура (Priority: P1)

- Как пользователь, я загружаю файл в формате `.glc`, смотрю превью контуров, нажимаю кнопку "Обробити" и получаю корректно обработанные контуры с применённым спецраскроем.

**Почему P1**: Это основная ценность — получение специального раскроя для производства.

**Independent Test**: Загрузить тестовый `.glc` с известными внутренними углами и проверить соответствие выходного `.glc` ожидаемому результату (см. Acceptance Scenarios).

**Acceptance Scenarios**:
1. Given: Пользователь загрузил валидный `.glc`; When: нажимает "Обробити"; Then: для каждого внутреннего угла выполнена одна из описанных трансформаций и контур модифицирован.
2. Given: Контур не содержит внутренних углов; When: нажимает "Обробити"; Then: контур остаётся неизменным и сохраняется в результирующий файл.

---

### User Story 2 - Просмотр и управляемый предпросмотр (Priority: P2)

- Как пользователь, я могу увидеть найденные контуры в SVG-превью, панить и зумировать колесом мыши, а после обработки увидеть обновлённое превью.

**Independent Test**: Открыть файл с несколькими контурами, проверить визуализацию и работу пан/зум; проверить, что превью обновляется после обработки.

---

### User Story 3 - Экспорт обработанного файла (Priority: P2)

- Как пользователь, я могу скачать новый `.glc` с суффиксом `_special_cut`, содержащий обработанные и не обработанные контуры.

**Independent Test**: После обработки нажать "Завантажити" и проверить, что в скачанном `.glc` присутствуют модифицированные контуры и имя файла изменено.

---

### Edge Cases

- Файл с некорректной геометрией — показать понятное сообщение об ошибке.
- Контуры с очень мелкими сегментами — применить правила пропуска (меньше 45 мм) и документировать это в журнале обработки.
- Перекрывающиеся контуры или многообъектные контейнеры: обработать только контуры в пределах одного объекта и не изменять метаданные других объектов.

## Требования *(обязательно)*

### Функциональные требования

- **FR-001**: Интерфейс должен позволять загружать `.glc` через кнопку и drag-and-drop.
- **FR-002**: Система должна распарсить `.glc` и обнаружить все контуры.
- **FR-003**: Опционально отображать все обнаруженные контуры в SVG-канве (пан/зум колесом).
- **FR-004**: По нажатию "Обробити" для каждого контура находить все внутренние углы (угол > 180° по методу: угол между предыдущим и следующим сегментами при обходе по часовой стрелке, угол измерять в общепринятом формате, принимая за начало отсчета направление предыдущего сегмента).
- **FR-005**: Обработку угла реализовать по правилам:
  - Если следующий за текущим угол внешний и расстояние до него < `COLUMN_DISTANCE_THRESHOLD` (по умолчанию 250 мм) и чекбокс «Не обробляти колони і простінки» установлен — пропустить оба угла.
  - Иначе, если один из прилегающих сегментов < `MIN_SKIP_LENGTH` (45 мм) — пропустить угол.
  - Иначе, если хотя бы один прилегающий сегмент < `SPLIT_THRESHOLD_SHORT` (110 мм) — разделить больший сегмент так, чтобы новый прилежащий сегмент стал равен меньшему; зафиксировать приращение как `SHORT_INCREMENT_PERCENT` (10%).
  - Иначе — разделить оба прилегающих сегмента так, чтобы рядом с углом появились отрезки длиной `NEAR_ANGLE_SEGMENT` (100 мм); зафиксировать приращение как `NEAR_INCREMENT_MM` (20 мм).
- **FR-006**: В последних двух случаях удлинить прилежащие к внутреннему углу отрезки на величину приращения и переместить вершину в точку пересечения окружностей (центры — дальние вершины соседних отрезков, радиусы = длина соответствующего отрезка + приращение).
- **FR-007**: Переименовать вершины контура начиная с `A` в направлении по часовой стрелке, чтобы имена были упорядочены.
- **FR-008**: Для каждого модифицированного контура установить параметры `StretchParamPer` и `StretchParamPer_2` равными `DEFAULT_STRETCH_PERCENT` (10%).
- **FR-009**: При нажатии "Завантажити" сформировать новый `.glc` со всеми контурами (обработанными и необработанными) и исходным именем + `_special_cut`.
- **FR-010**: Все пороговые величины и параметры должны быть вынесены в единый конфигурационный модуль/файл как константы/настройки.

### Критерии конфигурации (константы)

- `COLUMN_DISTANCE_THRESHOLD` = 250  # мм
- `MIN_SKIP_LENGTH` = 45  # мм
- `SPLIT_THRESHOLD_SHORT` = 110  # мм
- `NEAR_ANGLE_SEGMENT` = 100  # мм
- `SHORT_INCREMENT_PERCENT` = 10  # %
- `NEAR_INCREMENT_MM` = 20  # мм
- `DEFAULT_STRETCH_PERCENT` = 10  # %

## Сущности

- **Contour**: последовательность вершин и сегментов, ориентированная по обходу.
- **Vertex**: точка контура с координатами и именем (A, B, C...).
- **Segment**: отрезок между двумя Vertex с длиной и направлением.
- **ProcessingResult**: информация о том, какие вершины изменены, какие сегменты разделены и значения приращений.

## Критерии успешности *(обязательно, измеримые)*

- **SC-001**: 100% тестовых `.glc` с известными внутренними углами проходят обработку без ошибок (unit/e2e тесты).
- **SC-002**: 95% выявленных внутренних углов корректно модифицируются в соответствии с правилами на тестовом наборе (измеряется сравнениями геометрии).
- **SC-003**: Время обработки файла до 5 сек для файла с 50 контурами (базовая производительность для десктопа/серверного окружения).
- **SC-004**: Пользователь может скачать результат в формате `.glc` и имя файла содержит суффикс `_special_cut`.

## Assumptions / Предположения

- Единицы измерений — миллиметры (мм).
- Формат `.glc` уже описан в `.docs/GLC_FORMAL_SPECIFICATION` и парсер проекта способен предоставить список контуров и их вершин.
- Окружности для вычисления пересечения — допускается численная аппроксимация с погрешностью до 1 мм.
- UI локализация: интерфейс и документация будут на украинском языке.
